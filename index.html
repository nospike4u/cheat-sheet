Application Setup Cheatsheet

How to make an App as a Bootcamper

This sheet provides an overview of how to make an app as a Bootcamp. The Cheatsheet is a go to guide on topics designed to take you from beginning to end on the topics of: Conceptualising the idea, tools to use to help the design phase, structuring the project, tips on research, the development phase, testing and release and presenting. We know it is a lot but remember, this is the point of the cheatsheet; it is a go-to reminder to support your brain!

Technologies

Git is a distributed version control system that tracks versions of files. It is often used to control source code by programmers collaboratively developing software.


GitHub is a developer platform that allows developers to create, store, manage and share their code. It uses Git software, providing the distributed version control of Git plus access control, bug tracking, software feature requests, task management, continuous integration, and wikis for every project.


Vite, French word for "quick", is a build tool that aims to provide a faster and leaner development experience for modern web projects. Use this to install applications like React, Tailwind, Express and many more that suits your project needs.


Tailwind CSS is a utility-first CSS framework for rapidly building modern websites without ever leaving your HTML.


Bootstrap (formerly Twitter Bootstrap) is a free and open-source CSS framework directed at responsive, mobile-first front-end web development.



React (also known as React.js or ReactJS) is a free and open-source front-end JavaScript library for building user interfaces based on components.


Node.js is a cross-platform, open-source JavaScript runtime environment that can run on Windows, Linux, Unix, macOS, and more. Node.js runs on the V8 JavaScript engine, and executes JavaScript code outside a web browser.


MongoDB is a source-available, cross-platform, document-oriented database program. Classified as a NoSQL database product, MongoDB utilizes JSON-like documents with optional schemas.


SQLite is a database engine written in the C programming language. It is not a standalone app; rather, it is a library that software developers embed in their apps. As such, it belongs to the family of embedded databases.


PostgreSQL, also known as Postgres, is a free and open-source relational database management system emphasizing extensibility and SQL compliance.

Concept Phase

Concept phase By following these steps during the concept phase, the team ensures a well-defined roadmap for the project and a solid foundation for the development phase. Define Project Goals and Requirements:

  • Identify Objectives: Understand what the app aims to do.
  • Gather Requirements: Collect functional and non-functional requirements.
  • Brainstorm and Ideation:
  • Collaborate on Ideas: Hold a brainstorming session to generate ideas and features that the app will contain.
  • Prioritize Features: Set a priority for the features (Major, minor and nice to have)
  • Sketch Wireframes and Prototypes:
  • Create Wireframes: Develop low-fidelity sketches of the app's layout and key screens.
  • Build Prototypes: Use tools like Figma or Excalidraw to create prototypes. Plan the technologies that you will use:
  • Choose Tech Stack: Decide on the technologies to be used, such as React, Vite, and any necessary libraries or frameworks.
  • Define Architecture: Outline the app’s architecture, including component structure, state management, and backend integration (if needed). Develop Project Plan and Timeline:
  • Assign Tasks: Distribute tasks among team members based on their skills and expertise. (Remember to always get out of your comfort zone and try new things. You are here to learn)
  • Set Up Collaboration Tools: Project Management: Use tools like Trello for task management.
  • Version Control: Set up a version control system like GitHub and establish branching strategies.
  • Communication: Use communication tools like Slack for effective team collaboration.

Git Basic definitions

  • Local repo or repository: A local directory containing code and files for the project
  • Remote repository: An online version of the local repository hosted on services like GitHub, GitLab, and BitBucket
  • Cloning: The act of making a clone or copy of a repository in a new director. Commit: A snapshot of the project you can come back to
  • Branch: A copy of the project used for working in an isolated environment without affecting the main project
  • git merge: The process of combining two branches together.
  • .gitignore file: A file that lists other files you want git not to track (e.g., large data folders, private info, and any local files that shouldn’t be seen by the public).
  • Staging area: a cache that holds changes you want to commit next
  • git stash: another type of cache that holds unwanted changes you may want to come back later
  • Commit ID or hash: a unique identifier for each commit, used for switching to different save points.
  • HEAD (always capitalized letters): a reference name for the latest commit, to save you having to type Commit IDs. HEAD~n syntax is used to refer to older commits (e.g. HEAD~2 refers to the second-to-last commit).

Presentation Phase

  1. Introduction Keep It Short: Start with a brief introduction of your project. Team Introduction: Introduce all team members and their roles in the project.
  2. Project Overview High-Level Summary: Provide a high-level summary of the project. Mention the main features and functionalities. Technologies Used: Briefly mention that the project is built with React and any other key technologies.
  3. Task Allocation Splitting Tasks: Explain how the tasks were divided among team members. Highlight the areas of responsibility. Role-Specific Insights: Allow each team member to talk briefly about their part of the project and their contributions.
  4. Development Process Communication: Describe how you kept the communication flowing during the development process. Mention any tools (like Slack, Trello, or GitHub) that you used. Challenges and Solutions: Briefly touch on any challenges faced and how you overcame them as a team.
  5. Project Demonstration Focus on Results: Emphasize showing the final results rather than delving into the code. Demonstrate the working features of the project (avoid showing incomplete parts, missing functionality and such).
  6. Conclusion - Summary of Achievements: Summarize the key achievements of the project. Team Reflections: End with each team member sharing a quick reflection on what they learned and enjoyed about the project.
  7. Q&A Session - Prepare for Questions: Be ready to answer questions from colleagues and teachers. Ensure each team member is prepared to discuss their specific contributions.
